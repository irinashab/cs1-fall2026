Lesson: Strings & Typography in Processing

What is a String?

A String is a sequence of characters enclosed in quotes.


String s = "Hello";
String name = "Irina";

Strings are objects, not primitive types. That distinction matters.

⚠️ Why you CANNOT compare strings with ==

This is one of the most important conceptual points.


String a = "hello";
String b = "hello";

if (a == b) {
  println("Same");
}

This may appear to work sometimes — but it is unreliable.

What is actually happening?

  • == compares memory addresses
  • Strings are objects
  • So you are asking: are these the same object?

Instead, you want:


if (a.equals(b)) {
  println("Same content");
}

Correct mental model

  • == → identity (same object)
  • .equals() → content (same characters)

Rule: For strings, always use .equals().

Common String Operations


String s = "Processing";

println(s.length());     // 10
println(s.charAt(0));    // 'P'
println(s.substring(0,4)); // "Proc"

Typography in Processing

Strings become visual when used with typography functions.

Basic text


text("Hello", 100, 100);

Fonts


PFont f;

void setup() {
  size(400,400);
  f = createFont("Arial", 32);
  textFont(f);
}

Alignment


textAlign(CENTER, CENTER);
text("Centered", width/2, height/2);

Typography Examples (based on your sketches)

1. Letters in Motion

  • Each character moves independently
  • Requires splitting a string into characters

String word = "HELLO";

for (int i = 0; i < word.length(); i++) {
  char c = word.charAt(i);
  text(c, 50 + i*20, 100);
}

2. Opacity / Fade


fill(0, 150); // transparency
text("fade", 100, 100);

3. Interaction (hover / avoid)

  • Characters react to mouse
  • Position becomes dynamic

4. Bulge / Distortion

  • Typography reacts to distance
  • Size or spacing changes

Common Student Mistakes

  • Using == instead of .equals()
  • Forgetting Strings are objects
  • Trying to modify characters directly (Strings are immutable)
