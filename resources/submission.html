Programming Assignments: File Naming & Submission Guidelines

When submitting a programming assignment, please follow these guidelines.

1) File naming convention

  • Name each Processing sketch using the following format: firstInitialLastName_AssignmentName_Q#
  • For example: iShablinsky_Lab1_Q2 or iShablinsky_HW1_Q1
  • The .pde extension will be added automatically by Processing, so you do not need to type it.
  • Allowed characters: letters, numbers, and underscores (_). Do not use spaces or other special characters.

2) Top-of-file comments

Every sketch must begin with comments that include:

  • Your first and last name
  • Assignment name
  • Question number
  • Date completed

Example:

// Alan Turing
// Homework 3
// Question 2
// 01/23/1937

3) How to submit

  1. Locate your sketch folder (Processing menu: Sketch → Show Sketch Folder).
  2. Compress (zip) the entire sketch folder.
  3. Upload the .zip file to Brightspace.
Do not zip only the .pde file. Zip the entire sketch folder.
If you are submitting multiple sketches for a single assignment: Place all sketch folders inside a single parent folder. Compress (zip) the parent folder and submit the resulting .zip file.
Important: If your sketch uses images, fonts, sounds, or other external files, make sure they are stored in the sketch's data folder before creating the .zip file.
