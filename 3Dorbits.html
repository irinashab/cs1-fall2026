Lesson 17: Circular and Elliptic Motion in 2D and 3D

Big Idea

Circular and elliptic motion are both built from the same two functions: cos() and sin(). In Processing, the variable that changes over time is usually an angle or parameter such as angle or t.

In 2D, we usually write positions as (x, y). In 3D, we add one more coordinate: (x, y, z). A 3D orbit can simply be a 2D curve placed inside a plane in 3D space.

Part 1: Circular Motion in 2D

A point moving on a circle around a center (centerX, centerY) can be described by: 

float x = centerX + radius * cos(angle);
float y = centerY + radius * sin(angle);

As angle increases, the point moves around the circle. 

float centerX, centerY;
float radius = 120;
float angle = 0;
float speed = 0.02;

void setup() {
  size(600, 400);
  centerX = width / 2;
  centerY = height / 2;
}

void draw() {
  background(255);

  // Draw orbit path
  noFill();
  stroke(180);
  ellipse(centerX, centerY, 2 * radius, 2 * radius);

  // Compute moving object's position
  float x = centerX + radius * cos(angle);
  float y = centerY + radius * sin(angle);

  // Draw center
  fill(0);
  noStroke();
  ellipse(centerX, centerY, 10, 10);

  // Draw orbiting object
  fill(50, 100, 255);
  ellipse(x, y, 30, 30);

  angle += speed;
}

Part 2: Circular Motion with a Trail

To leave a trail, do not clear the screen completely with background() each frame. Instead, draw a semi-transparent rectangle over the whole canvas. The old frames fade gradually. 

fill(255, 20);
rect(0, 0, width, height);

Smaller alpha values make longer trails. Larger alpha values make shorter trails. 

float centerX, centerY;
float radius = 120;
float angle = 0;
float speed = 0.03;

void setup() {
  size(600, 400);
  centerX = width / 2;
  centerY = height / 2;
  background(255);   // only once
}

void draw() {
  // Fade previous frames
  noStroke();
  fill(255, 20);
  rect(0, 0, width, height);

  float x = centerX + radius * cos(angle);
  float y = centerY + radius * sin(angle);

  fill(50, 100, 255);
  ellipse(x, y, 20, 20);

  angle += speed;
}

Part 3: Orbit and Self-Rotation in 2D

There are two different kinds of rotation:

  • Orbit: the object moves around another object.
  • Self-rotation: the object spins around its own center.

In Processing, the order of transformations matters. This is the key sequence: 

translate(cx, cy);          // go to the center object
rotate(orbitAngle);        // rotate around the center
translate(orbitRadius, 0); // move outward
rotate(selfAngle);         // spin around local center
float cx, cy;
float orbitRadius = 120;
float orbitAngle = 0;
float selfAngle = 0;

void setup() {
  size(600, 400);
  cx = width / 2;
  cy = height / 2;
  rectMode(CENTER);
}

void draw() {
  background(255);

  // central object
  fill(0);
  noStroke();
  ellipse(cx, cy, 20, 20);

  // orbit path
  noFill();
  stroke(180);
  ellipse(cx, cy, orbitRadius * 2, orbitRadius * 2);

  pushMatrix();
  translate(cx, cy);
  rotate(orbitAngle);
  translate(orbitRadius, 0);
  rotate(selfAngle);

  // rectangle makes self-rotation visible
  fill(255, 150, 50);
  noStroke();
  rect(0, 0, 60, 20);

  // marker dot
  fill(50, 100, 255);
  ellipse(20, 0, 10, 10);

  popMatrix();

  orbitAngle += 0.02;
  selfAngle += 0.08;
}

Part 4: Circular Motion of a Sphere in 3D

To work in 3D, use P3D in size(). A circular orbit can live in the xz-plane by changing x and z, while keeping y fixed. 

float x = radius * cos(angle);
float z = radius * sin(angle);
float radius = 150;
float angle = 0;

void setup() {
  size(800, 600, P3D);
}

void draw() {
  background(255);
  lights();

  translate(width / 2, height / 2, 0);
  rotateX(PI / 5);  // tilt so we can see depth

  // center sphere
  fill(0);
  noStroke();
  sphere(20);

  // orbit path in xz-plane
  noFill();
  stroke(180);
  beginShape();
  for (float a = 0; a < TWO_PI; a += 0.05) {
    float x = radius * cos(a);
    float z = radius * sin(a);
    vertex(x, 0, z);
  }
  endShape(CLOSE);

  // moving sphere
  float x = radius * cos(angle);
  float z = radius * sin(angle);

  pushMatrix();
  translate(x, 0, z);
  noStroke();
  fill(80, 140, 255);
  sphere(25);
  popMatrix();

  angle += 0.02;
}

Part 5: 3D Orbit and Self-Rotation

A plain sphere is symmetric, so its self-rotation is hard to see. Add a small marker sphere on the surface to make the spin visible. 

float orbitAngle = 0;
float selfAngle = 0;
float orbitRadius = 150;

void setup() {
  size(800, 600, P3D);
  sphereDetail(30);
}

void draw() {
  background(255);
  lights();

  translate(width / 2, height / 2, 0);
  rotateX(PI / 5);

  // center sphere
  fill(50);
  noStroke();
  sphere(25);

  // orbit path
  noFill();
  stroke(180);
  beginShape();
  for (float a = 0; a < TWO_PI; a += 0.05) {
    float x = orbitRadius * cos(a);
    float z = orbitRadius * sin(a);
    vertex(x, 0, z);
  }
  endShape(CLOSE);

  pushMatrix();

  // orbit around center in the xz-plane
  rotateY(orbitAngle);
  translate(orbitRadius, 0, 0);

  // self-rotation around local y-axis
  rotateY(selfAngle);

  // main sphere
  fill(100, 150, 255);
  noStroke();
  sphere(35);

  // marker on surface
  pushMatrix();
  translate(35, 0, 0);
  fill(255, 100, 100);
  sphere(8);
  popMatrix();

  popMatrix();

  orbitAngle += 0.02;
  selfAngle += 0.08;
}

Part 6: Elliptic Motion Without Polar Coordinates

An ellipse can be understood as a stretched circle. Start with circular motion: 

cos(t), sin(t)

Then stretch it differently in the horizontal and vertical directions: 

float x = centerX + a * cos(t);
float y = centerY + b * sin(t);

Here a is the horizontal radius and b is the vertical radius. This is parametric motion, not polar coordinates. 

float cx, cy;
float a = 200;
float b = 120;
float t = 0;

void setup() {
  size(600, 400);
  cx = width / 2;
  cy = height / 2;
}

void draw() {
  background(255);

  // ellipse path
  noFill();
  stroke(180);
  ellipse(cx, cy, 2 * a, 2 * b);

  // moving object
  float x = cx + a * cos(t);
  float y = cy + b * sin(t);

  fill(50, 100, 255);
  noStroke();
  ellipse(x, y, 20, 20);

  t += 0.02;
}

Important note: this does not move at constant speed along the ellipse. The parameter t changes uniformly, but the distance traveled per frame changes.

Part 7: Elliptic Motion in 3D

To add the third coordinate, place the ellipse into one of the coordinate planes. For example, an ellipse in the xz-plane has: 

float x = a * cos(t);
float y = 0;
float z = b * sin(t);
float a = 220;
float b = 120;
float t = 0;

void setup() {
  size(800, 600, P3D);
}

void draw() {
  background(255);
  lights();

  translate(width / 2, height / 2, 0);
  rotateX(PI / 5);

  // ellipse path in xz-plane
  noFill();
  stroke(180);
  beginShape();
  for (float u = 0; u < TWO_PI; u += 0.05) {
    float x = a * cos(u);
    float z = b * sin(u);
    vertex(x, 0, z);
  }
  endShape(CLOSE);

  // moving sphere
  float x = a * cos(t);
  float z = b * sin(t);

  pushMatrix();
  translate(x, 0, z);
  fill(100, 150, 255);
  noStroke();
  sphere(20);
  popMatrix();

  t += 0.02;
}

Part 8: Ellipse in a Tilted Plane

Another way to make a 3D orbit is to create a flat ellipse first, then rotate the whole plane. This keeps the motion simple while making the orbit look three-dimensional. 

float a = 220;
float b = 120;
float t = 0;

void setup() {
  size(800, 600, P3D);
}

void draw() {
  background(255);
  lights();

  translate(width / 2, height / 2, 0);

  // Tilt the entire orbital plane
  rotateX(PI / 4);
  rotateY(PI / 6);

  // ellipse path in local xy-plane
  noFill();
  stroke(180);
  beginShape();
  for (float u = 0; u < TWO_PI; u += 0.05) {
    float x = a * cos(u);
    float y = b * sin(u);
    vertex(x, y, 0);
  }
  endShape(CLOSE);

  // moving sphere
  float x = a * cos(t);
  float y = b * sin(t);

  pushMatrix();
  translate(x, y, 0);
  fill(100, 150, 255);
  noStroke();
  sphere(20);
  popMatrix();

  t += 0.02;
}

Summary

  • Circle: use the same radius for both directions.
  • Ellipse: use different stretch factors, a and b.
  • 3D orbit: add a third coordinate and place the curve in a plane.
  • Tilted orbit: draw a simple flat orbit, then rotate its coordinate system.
  • Self-rotation: move to the object first, then rotate locally.
