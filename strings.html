Lesson: Strings & Typography in Processing

What is a String?

A String is a sequence of characters enclosed in quotes.

String s = "Hello";
String name = "Irina";

In Java/Processing, a String is an object, not a primitive type like int or float. That distinction matters when we compare strings, pass them to methods, or try to modify them.

Why we cannot compare strings directly with ==

This is one of the most important points in the whole lesson.

When you write == with strings, you are not asking whether the two strings contain the same text. You are asking whether they are the same object in memory.

String a = "hello";
String b = "hello";

println(a == b);

This may print true. But that does not mean == is correct for strings.

Sometimes the compiler notices identical constant strings and reuses the same object. That can trick you into thinking direct comparison works.

A misleading example

String HELLO = "Hello";
String hello = "Hello";

println(hello == HELLO);

This may print true because both variables may point to the same stored string object.

But now look at this

String HELLO = "Hello";
String hell = "Hell";
String hello = hell + "o";

println(hello == HELLO);

Now the result is typically false, even though both strings contain exactly the same characters.

Why? Because hello was built during the program, so it is a different object, even though its text is still "Hello".

The correct way

String HELLO = "Hello";
String hell = "Hell";
String hello = hell + "o";

println(hello.equals(HELLO));

This checks the contents of the strings, which is what we usually mean when we ask whether two strings are equal.

Correct mental model

  • == asks: are these the same object?
  • .equals() asks: do these objects contain the same text?

Rule: if you want to compare strings, use .equals(), not ==.

Useful String operations

String s = "Processing";

println(s.length());        // number of characters
println(s.charAt(0));       // character at index 0
println(s.substring(0, 4)); // "Proc"

These methods do not change the original string. They either return information or create a new string.

Strings are immutable

Immutable means that once a String object is created, its contents cannot be changed.

For example, suppose we start with:

String word = "cat";

You cannot directly change just one character inside that string object. There is no command that turns the existing "cat" object into "cut".

When it looks like you are “changing” a string, what actually happens is that you create a new string and usually store it back into the same variable.

String word = "cat";
word = "cut";

The variable word now refers to a different string object. The original "cat" string was not edited in place.

What if you need to fix a typo?

You build a new string.

For example, suppose you have:

String typo = "Helo";

One simple fix is just reassignment:

typo = "Hello";

But sometimes you want to fix the typo systematically. Then you can create a new string using methods such as substring(), concatenation, or replace().

Fixing with concatenation and substring

String typo = "Helo";
String fixed = typo.substring(0, 3) + "l" + typo.substring(3);

println(fixed);

This does not repair the old string object. It creates a new one.

Fixing with replace()

String typo = "I liek Processing";
String fixed = typo.replace("liek", "like");

println(fixed);

Again, replace() returns a new string. The old one stays unchanged unless you assign the result to a variable.

String typo = "I liek Processing";
typo.replace("liek", "like");
println(typo);

The typo is still there, because the result was not saved.

typo = typo.replace("liek", "like");
println(typo);

Now the variable points to the corrected version.

Typography in Processing

Strings become visual when we draw them with Processing.

Basic text

text("Hello", 100, 100);

Fonts

PFont f;

void setup() {
  size(400, 400);
  f = createFont("Arial", 32);
  textFont(f);
}

Alignment

textAlign(CENTER, CENTER);
text("Centered", width/2, height/2);

Typography example: Flexibility

In the sketch flexibility.pde, the word is turned into an array of characters so each letter can be handled separately.

String word = "Flexibility";
char[] letters;

void setup() {
  letters = word.toCharArray();
}

The method toCharArray() extracts the individual characters from the string and places them into a character array.

That is useful because each letter can now be drawn one by one, rotated, spaced, or animated independently.

for (int i = 0; i < letters.length; i++) {
  float angle = map(mouseX, 0, width, 0, PI / 8);
  rotate(angle);
  text(letters[i], 0, 0);
  translate(textWidth(letters[i]), 0);
}

The string itself is still immutable. We are not changing the original word. We are using its characters as drawing data.

Common student mistakes

  • Using == instead of .equals()
  • Thinking a string variable stores “just text” rather than a reference to an object
  • Assuming methods like replace() permanently change the original string
  • Forgetting that if you want the updated text, you must save the returned string
